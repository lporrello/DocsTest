Per-Company Facebook Apps
Prior to the 4.05 release (April 2012), the vast majority of companies configured in the system published to Facebook using ’s own Facebook application. Exceptions were white label clients who were given a custom branded application—this appears as the “via company name” text that follows a post on Facebook.over time, using a shared Facebook app became problematic; one company’s “spamminess” could impact the rate limits imposed on other companies, for example. Post 4.05, each new company can be configured using its own Facebook app. Before creating a company, you must first obtain a Facebook App Name, App ID, and App Secret. To obtain a Facebook API key and secret, go to the Facebook Developer App. Click Set Up New Application. Provide a name and agree to the terms. For Connect URL, enter http://[your primary domain]. The next page provides the API key and secret.

Create a New Company in :

  1. From the Admin Panel, Company menu, click Companies.
  2. Click New Company.
  3. In the Name field, enter the company name. See Parent Company for more information.
  4. If the new company is associated with (a “child of”) a parent company, select the parent company from the drop-down menu.
  5. Provide the Facebook App Name, App ID, and Secret. If these fields are left blank, the Facebook app will be used. If you complete one of these fields, all three become required in order to create the company.
  6. Click Save.

When authenticating an account, the Facebook App configured here is shown as the “default” app. If additional branded applications are created for the company, users must select the one to use when authenticating an account. See Branded Applications.

Create and Manage Assets

An asset can contain a piece of text, an image, or both. The text asset size limit is around 4,000 characters. The file size limit is 3 MB. Supported file types are JPG, GIF, and PNG. Assets are stored in folders. but you can see all the assets you have access to at once by expanding the Folder panel and clicking the All Assets folder at the top of the list.

Each asset in the Content Center is represented as an individual panel. If the asset contains both an image and text, a Txt icon appears on the upper right-hand side of the image; rollover it to see the associated text. If the asset contains only text, the text is shown in the panel. When you click on an asset, a dialog containing all information about the asset appears. The properties associated with each asset are defined in the table below.

Content Center Asset Properties

Property

Definition

On Asset thumbnail

Name

Internal name for the asset.

Used

The total number of times the asset has been included in a message scheduled to be published. (See Total and Network Usage, below)

Created (date)

The date the asset was added to the Content Center

Characters (count)

The number of characters the text asset (or text portion of the asset) contains.

In Asset Property dialog

Name

Click the Edit (pen icon) to change the name.

Location

The folder containing the asset.

Network Usage

The number of times the asset has been included in a message that has been put on the Calendar (including Draft status), by social channel. If the message is published to multiple accounts, each one is counted separately. Reposts and Series messages are included If no account has been selected yet (for example, if the message is a draft), usage is counted as 1 until account(s) are chosen. There may be up to a 5 minute delay before data from newly create messages appear in the count.

Total

The total number of times the asset has been included in a message that has been put in the Calendar.

Labels

Outbound labels applied to the asset. While label application is not scoped, only labels available in the active Initiative appear in the picker menu. If a label does not appear as expected, ensure that it is available in the current Initiative.

Start Date

If the asset can be published only for a limited time period (such as a holiday season) or if use of the asset must be prohibited until a particular date (a product release, for example), set the start and end dates. An asset cannot be published before a start date, or after the end date.

Expiration date

If the asset can be published only for a limited time period (such as a holiday season) or if use of the asset must be prohibited until a particular date (a product release, for example), set the start and end dates. An asset cannot be published before a start date, or after the end date. After the end date, the item cannot be selected for publishing.

Suitable for

Identify the social channels the asset is appropriate for. For example, longer text messages would not be suitable for Twitter, but could be used on Facebook or LinkedIn. This categorization helps users find an appropriate asset in the Content Center, but is not restrictive; does not enforce these categories when a message is scheduled for publishing.

Intended Use

Publication means the asset is appropriate to post to one of the social channels. Moderation means the asset is appropriate to use to respond to items that appear in the Inbox. This categorization helps users find an appropriate asset in the Content Center, but is not restrictive; you can use Publication assets from the Inbox, or publish Moderation assets.

Creator

The user who originally added the asset to the Content Center.

Description

Text about the asset.

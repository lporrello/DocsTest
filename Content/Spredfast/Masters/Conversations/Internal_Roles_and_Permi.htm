Internal Roles and Permissions

Internal Roles vs. Company Roles
This section describes the roles and permissions that exist within the system for administering . These must not be confused with customer roles and permissions that are assigned to end users within a customer company. Internal roles can be edited using Change User Role on the Admin Panel. Customer roles can be edited from the Users tab in My Company.

Admin ()—(for PM, DEV, CST, QA, OPS, SE, No customers):

  • All Admin Panel functions, including Impersonate.
  • Can create custom user roles (company roles) in areas other than Content Center and Promotion. At one time, customers could create their own; now they can only be created by Admins, typically by Support.

User—(Customers): No Admin Panel access.

Sales—( sales reps for demo purposes): Admin Panel (Companies, Users, Edit)

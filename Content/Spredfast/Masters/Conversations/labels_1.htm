Account

The Account pull down control shows a list of Account Sets and accounts that you can use to post the message.

Note that the list includes only those accounts for which you (the message creator) have permission to publish. If you have access to only one account in the channel, it is pre-selected. If a different person (someone other than you) is the assignee for the message, the list contains only accounts for which both of you have permission to publish.

If the selected account requires an approval path because of the Account Set it is in, the Workflow tab is updated with the number of approvals required. (For more information about approval paths, see About Approvals.)

Link Preview (Facebook, Google+, LinkedIn)

Link Preview, Photo, or Video
On Facebook and Google+ messages, you can include only one type of link preview.

After you type a link in the message, obtains web page data (title, description, and images) and provides it as a preview in the message.

One Link Preview per Message
Though you can embed multiple links in a post, only one can have a link preview. By default, the system uses the first link; you can change by selecting Attach link preview on a different link.

 

For Facebook and LinkedIn—You can edit both the title and description before you publish; click on the Edit button (pen icon) to bring up an editable field. If there are multiple image files, you can choose among them by clicking the left/right arrows that appear when you hover on the image. Eliminate the thumbnail by clicking the X icon in the upper right hand corner of the image; return it by clicking Show Photo. To upload a custom image, click in the Add a Photo pane, and, in the Add a Photo From dialog, choose either Computer or Content Center.

For Google+—The title and description fields cannot be edited. You can change the thumbnail image from among the ones provided, but you cannot turn off the thumbnail or supply your own.

To delete the link preview from the message, click the down arrow next to the link in the message and deselect Attach Link Preview.

Information that appears in the LinkedIn and Facebook Link Preview is obtained from embed.ly.

Information that appears in the Google+ Link Preview is obtained from Google.

Link Creation and Shortening

Within microblogs, is able to determine if message text you enter is a link. When link text is recognized, it is automatically tokenized and shortened when you publish the post.

Automatic link shortening is enabled on a company-wide basis. It is turned ON by default but can be turned OFF in the Admin Panel if a company requests. See Per-Company Configurations—Admin Panel.

How does determine if a piece of text is a link? We use the same link-detection algorithm as Twitter.

Once the link text is tokenized, you can do the following:

  • Choose NOT to shorten the link—deselect Shorten on Publish. Note that wraps all links with a t.co url, regardless of this setting. By default, uses our Bitly account to shorten links, but if a different one has been configured for the Initiative (Initiative Settings > Link Shortening), that one is used instead.

Automatic Link Shortening
For the vast majority of organizations using , link shortening is performed automatically. If your organization has requested that this feature be disabled, the Shorten on Publish link is already deselected. For more information on disabling this feature, contact your Account Manager or Support.

Link Shortening and Kenshoo Integration
When Kenshoo tagging is used for an Initiative, all links published through that Initiative must be shortened; users will not have the option to deselect Shorten on Publish when adding links to messages.
For more information, see Third-Party Apps.

  • Include a link preview (Facebook, Google+, LinkedIn)—Attach link preview. Title, description, and image information obtained from the linked page are inserted in the message.
  • Test the link in a new window—Open link.
  • If you have integrated with Google Analytics or Omniture accounts, you can edit the embedded link tags, either adding new ones or editing the default ones. If embedded link tags have been configured, an additional menu is made available; click Edit Link Tags.
  • Note that if the text is identified as being in a link format, the link will always be made. If you don’t want the text to become a link, you must rewrite it, for example refer to “the Google website” rather than “Google.com.”

Photo or Video

You can attach supported media to a microblog message on Facebook, Twitter, Google+, and Weibo. In the Media field, choose either Computer and upload from your desktop, or Content Center. Only one photo or video is allowed per message. Supported photo file types are jpeg, gif, png; size limit is 3MB. Supported video file types include g2, 3gp, 3gpp, asf, avi, dat, divx, dv, f4v, flv, m2ts, m4v, mkv, mod, mov, mp4, mpe, mpeg, mpeg4, mpg, mts, nsv, ogm, ogv, qt, tod, ts, vob, wmv; size limit is 5 GB.

Facebook Video Call to Action

You can configure your Facebook video posts so that at the end of each video the viewer will see a call to action with an accompanying URL.

The Call to Action pull down control enables you choose the message that appears in the button that appears at the end of the video in the Facebook mobile app and the overlay that appears at the end of a video in the Facebook computer app.

The Call to Action Link URL field is where you will enter the URL that you want viewers to click.

Content Center

To Include assets from the Content Center, in the Media field, click the Content Center icon. In the Select Assets screen, click the Folder Panel button to see available assets, and click on the item. If the selected content includes link text, the link is shortened and tokenized. The Content Center icon appears only if 1) the Content Center has been enabled for your company, 2) The Content Center is in your Workspace and 3) You have permission to publish from the folder in which the item is stored, though the folders will not be visible to you when assets are shown. If the channel you are publishing to does not support attached images, you can access the Content Center, but will see only “Text” assets.

For more information, see Content Center.

Schedule

Publish Now—Is always selected by default. If you do not use the Schedule control to choose a schedule, the message will publish as soon as all required approvals have been made. If no approvals are needed, or if they are made quickly, the message can publish within the five minute interval in which it was created. If approvals take longer than five minutes, the message publishes at the next five minute interval after the final approval was given. Also, whenever a “publish as soon as possible” message is edited, the time listed in the agenda is updated to reflect the time it was most recently saved (and consequently, ASAP means as soon as possible after the latest save).

Schedule—Select a date and time for the message to publish. You can schedule using the Calendar menu or type in terms such as “Next Thursday” or “Wednesday the 3rd” and the platform applies the correct date. If all required approvals are made prior to the scheduled date and time, the message posts at the scheduled time (within a five minute window). If approvals aren’t provided until after the scheduled time, the message will post in the next five-minute interval after the last approval is received. To remove a Schedule and return the message publishing time to As Soon as Possible, click the X. next to the scheduled date and time.

Repost—(Note that this refers to the Repost publishing feature, not the Weibo repost available from the Inbox) allows you to post the identical content, on the same accounts, at a later date and time. You cannot repost within 24 hours (plus five minutes) of the original post, reinforcing Twitter/Facebook rules and good social practices. Once the original post is published, you can no longer edit the content, accounts, path, or assignee of a reposted message, nor can you switch the scheduled time of a repost back to Publish as soon as Possible.

Time zone—The time zone shown at the upper right of the Schedule field is the one set in the My Account screen. ([Your name] > My Account). See Time Zone.

Planing projects with multiple time zones
If you’re planning a project that spans multiple time zones, remember that shows information to each user in his own time zone. Always plan posting based on what the desired time will be is in your time zone.
For example. you are in Austin, Texas, on Central Time, and want a message to go out at 8 a.m. in New York City, which is Eastern time. You schedule the message to post at 7 a.m. (in Central Time, which is 8 Eastern). Your colleague who works in Los Angeles will see the item scheduled on the Agenda for 5 a.m.

Copy Message—Once you’ve created a message, Copy Message creates a new message of the same type, with all the information from the original message pre-populated in the new one. Messages are not associated or linked to each other in any way; content from original just prepopulates the copy.

Labels

In the Labels field, use the Add labels drop-down to add a label to a message. Click in the field, and begin to type a label. If the label is already present in and the active Initiative has permission, it appears in the field; select it to add it to the message. Labels that have been designated as “top labels” appear in bold within the list. Some users are able to create new labels from this screen. Type the new label in the field and click [Enter]; the new label appears in the list and is available for others in the company to use.

Notes & Activity

Add internally facing Notes to messages (microblogs) for other users of to read. These are separate from the Notes used on Stream Items in the Inbox. Once saved, notes cannot be edited or deleted. Maximum length is 500 characters per note. You must save the message (Save Draft or Publish on the message creation page) to save a note. Once a note has been added to the message, all users in the Initiative can see it; users with permission to edit the message can add notes to it.

The Notes & Activity pane displays publishing actions taken on the message within . It provides the date, user, and the following actions: create, edit, submitted, approved, rejected, resubmitted.

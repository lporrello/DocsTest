Per-Company Configurations—Admin Panel

These settings are enabled on a per-customer basis. In the Admin panel, Companies page.

  1. Find the company using the search feature.
  2. Click the Company name.
  3. Click Edit Details. A dialog will appear featuring the following options.

Bazaarvoice Enabled

This item is OFF by default. Enables Company integration with the Bazaarvoice API, which allows creation of Bazaarvoice review streams and amplification of Bazaarvoice reviews. You must provide a Key (Customer-specific Bazaarvoice API KEY, obtained from Bazaarvoice specifically for the integration; this is different from the key customers have received from Bazaarvoice for their own purposes) and a URL (Customer-specific API endpoint). The URL must be fully specified (using http://), and is validated upon import. If new Keys/ URLs added later, they override previously supplied ones. Only one URL/KEY combination is supported per company. If the customer has chosen to allow only Company Admins to add new labels, then the label AMPLIFY_BV must be created for the company so that it can be automatically appended to amplified content.

Google+ Personal Account Enabled

This item is OFF by default. When enabled, allows users to authenticate personal Google+ accounts in . For authenticated personal accounts, auto-imports posts, displays them on the Calendar, and collects metrics for them, which are aggregated with the other Google+ (Company page) data. No publishing to personal accounts is supported, and streams cannot be created in the Inbox.

Analytics 2.0

Do Not Change These Settings
You must have specific permission from Product Management to enable Analytics 2.0 for any customer.

This item is OFF by default. Enables integration of analytics data with a third-party for rendering and reporting. When enabled, all of the Company-level analytics dashboards are enabled, as well as any custom dashboards created by Services.

Enable for a Company:

  1. Obtain permission from Product Management.
  2. From the Admin Panel, click Companies.
  3. Locate the company in the list, and click Edit Details.
  4. In the Analytics 2.0 Data Enabled field, click Yes. Data begins to be sent to the analytics platform. Depending on the amount of data available for the company, this can take up to couple of hours.
  5. When the data is available, Analytics button in the masthead converts to a drop-down menu for viewers who are admins.
  6. Confirm presence of data in Message Explorers and Social Performance Dashboards. When the display and data content is acceptable, you can enable the feature for customers.
  7. In the Analytics 2.0 Admin Only/All Usersfield, select All Users. The Analytics drop-down becomes available to customer users.

Do not adjust the following settings without specific DEV permission:

  • Analytics 2.0 Force Data Sync: Runs a full ETL (Extract/Transform/Load) data sync. Demo Project allows addition of a second analytics project to the UI without other integration; used for demo purposes.

Shorten Links by Default

This item is ON by default. When set to OFF, links published through are not automatically shortened before publishing. If the customer has Kenshoo Enabled, you cannot turn this setting OFF, because links must be shortened for the Kenshoo tags to be added. This setting does not affect messages sent from the Mobile app; those links are always shortened. See Compose.

Message Delegation Enabled

This item is OFF by default. The message delegation feature allows a company to use a “hub and spoke” model for microblog message creation and publishing, instead of the default “centralized” model. It affects both message creation and approval path functionality.

In the default state (OFF, or the “centralized” model), a single instance of a micro-blog message is created, follows the approval paths configured for the Account Sets associated with it, and must receive approval from all associated Account Sets in order to be published through any of the Account Sets.

When enabled (turned ON, or the “hub and spoke” model), multiple instances of a message are created, one for each Account Set, and the message must receive approval only from the approval teams associated with that individual Account Set before being published. The instance of the message can be published as it was written, edited, or rejected regardless of what action is taken on the other instances of the message within the other Account Sets.

For more information, see About Approvals.

This item is OFF by default. This setting must be enabled (set to YES) for any customer receiving SFTP export data or SMTP (email) export. This feature allows collection of third-party comments data for Facebook photos that have been published through .

Grant SalesForce

This item is OFF by default. Enables integration with SalesForce. To integrate, the customer must have an enterprise account with SalesForce. A single login, password, and security token are used for the integration, which uses the web services API. Integration is enabled company--wide, and for all Initiatives. See SalesForce.

Kenshoo Enabled

This item is OFF by default. Enables integration with Kenshoo, so that links published through for selected Initiatives will be wrapped in proprietary Kenshoo tags. To integrate, the customer must have an account with Kenshoo, and access to the Kenshoo Server Number (KS#) and Kenshoo profile (profile#) values. See Third-Party Apps.

Promoted Content Enabled

This item is OFF by default. Enables paid promotion of selected Facebook messages to the Newsfeed in published Facebook Pages. See Content Promotion and Promotion. Note that when this setting is changed to OFF for a customer that had been using the feature, all components of the feature are immediately removed from the user interface. Status messages for running promotions will be hidden, and all credential mapping (accounts to pages, etc.) will be hidden--when resumed, it become visible, but users must re-authenticate the applicable ad accounts.

App Builder Enabled

This item is OFF by default. Once paid for and enabled for an organization, allows all users access to an app/promotion builder, located in Tools > App Builder from the masthead. Create contests, sweepstakes, and other promotions on Facebook and other channels. You must provide a Company Name. See App Builder.

App Builder

Partner Applications

These items are OFF by default. For customers who use partner applications that have access to Publishing and Content Center APIs. Once the partner application has been configured, it appears in this list as available.

Sysomos Activation

Sysomos accounts and streams are created through the back end. The customer should verify that the Sysomos API is enabled for them. They need to retrieve their token and its expiration date from their Sysomos Account Manager (or Enterprise Client Success Manager). Direct the customer to file a ticket with Support.

In addition, Support needs to gather the Company ID, Initiative ID and Account Set ID (persona_id), and the Sysomos token, user name and heartbeat ID. Using this information, Support runs a zfcli commands and initiates an Ops ticket. Support follows up with the customer when this process is complete.

Enable Sysomos

  1. From the Admin Panel, Miscellaneous menu, click Create Listening Stream.
  2. Use the Vendor drop-down to select Sysomos.
  3. Under Vendor Shared Fields, Enter the Spredfast Campaign (Initiative) ID and Persona (Account Set) ID in the appropriate fields.
  4. Enter the authentication token from the vendor in the Token field.
  5. Enter the username in the Vendor’s platform in the Username field.
  6. Under Sysomos Specific Fields, enter the Sysomos Heartbeat ID.
  7. Click Submit.

Suspend an Initiative

You must suspend an Initiative before you can delete it.

  1. From the Admin panel, find the company entry.
  2. In the Initiatives section, (scroll down beneath the Users) locate the Initiative and click Suspend. The Initiative displays with a red background.
  3. To reactivate the Initiative, click the Activate link.

Create and Manage Folders

All Assets in the Content Center are organized into folders. The Folder Owner initially decides who can see and act upon the assets within it. When it’s first created (by default), the folder and its contents can be seen and acted upon by the user who added it. A folder can be shared with all the users in your Company, all the users in a particular Initiative, or with individual (named) users. When sharing a folder, the Folder Owner decides the type of access others have.

Folder sharing rules can be assigned when the folder is first created, or at a later time, using the Share icon for the [Folder name] > Share Folder option in the action bar. Folder sharing rules can be subsequently changed by other users with the correct permissions.

Folder Sharing and User Permissions
Folder sharing is used to limit, not extend, a user’s abilities in the Content Center. For example...
If a user’s role allows him to publish content, but he has been given only Can View access to a folder, he cannot publish assets from that folder.
If a user’s role does not allow him to publish content, but he is given Can Publish access to a folder, he will not be allowed to publish assets from the folder.

Sharing for Initiatives, Companies, and Named Users
If a user has access to a folder as a member of an Initiative or company, and as a named individual, whichever access gives her the most capabilities is used. For example, if you have Can View access as a member of an Initiative, but have Can Publish access as an individual, you are allowed to publish assets from the folder.

Company members can be permitted to do the following:

  • See and download assets (View)
  • Publish assets (Publish). (Remember that a user must also have a role that allows publishing, plus access to the account in order to publish.)

Initiative Members Can be permitted to do the following:

  • See and download assets (View)
  • Publish assets (Publish). (Remember that a user must also have a role that allows publishing, plus access to the account in order to publish.)

Individual (named) users can be permitted to do the following:

  • Add new assets to the folder (Create)
  • Edit asset content or properties (Edit)
  • Publish assets (Publish) (Remember that a user must also have publishing and content center roles that allows publishing, plus access to the account in order to publish.)
  • See and download assets (View)
  • These privileges are hierarchical--if a user has Create, for example, he also has all the privileges listed beneath (Edit, Publish, and View).

Create a Folder in the Content Center

  1. From the , click Content.
  2. (Optional) Expand the Folder panel to see existing folders.
  3. From the header, click Create > New Folder.
  4. In the Folder Name field, provide a name.
  5. In the Folder Description field, summarize the intention for folder content.
  6. In the Share with field, start typing the name of a person or Initiative; the field will auto-complete with names of your Company, the Initiatives you have access to, and users in the Company.
  7. For each Initiative and user you have selected, assign the level of access within the folder.
  8. Click Create Folder.

Edit a Folder (Properties)

  1. From the click Content.
  2. Expand the Folder panel and click on the folder.
  3. From the folder’s name at the top of the screen, click the adjacent drop-down menu (chevron icon), and select Edit Folder.
  4. In the Edit Folder screen, you can change the following:
    • the Folder’s name
    • the Folder’s description
    • Click Save Changes.
  5. To share a folder (or change folder sharing rules):
  6. From the click Content.
  7. Expand the Folder panel and select the folder.
  8. From the folder’s name at the top of the screen, click the adjacent drop-down menu (chevron icon), and select Share folder.
  9. In the Who has access field, you can change the level of access for users and Initiatives, or remove access to the folder. If access is removed, the users (or Initiative members) will not be able to see the folder the next time they log in. If they had scheduled messages to publish from the folder, the publishing will complete as scheduled.
  10. In the Share with field, start typing the name of a person or Initiative name; the field will auto-complete with names of Initiatives you have access to, and other users in the Company. Your current company appears at top of list--if you want to share with entire company, select it.
  11. In the Who has access panel, for each Initiative and user you have selected, assign the level of access within the folder.
  12. Click Create Folder.

Delete a Folder

  1. From the click Content.
  2. Expand the Folder panel and select the folder.
  3. From the folder’s name at the top of the screen, click the adjacent drop-down menu (chevron icon), and select Delete Folder.
  4. In the confirmation dialog, click Delete.

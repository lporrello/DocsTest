Labels

Create labels to tag content, track its performance, and associate it with an Initiative or campaign.

Labels can be assigned to the whole company or individual Initiatives. By default, labels are assigned to the whole company.

From the Labels tab, click Assign Labels to Initiative button. When labels are assigned to Initiatives, they only show up in the labels picker for Messages and Stream Items if the users are in an Initiative the label is assigned to. All labels, regardless of scope, appear in the filters for Analytics and the Content Center. Returning the label scope to company-wide disables the scoping, but does not delete any Initiative scoping settings.

Case Sensitivity and Duplicity
Labels are not case sensitive. “Open” and “open” are interpreted to be the same label within . Additionally, labels must be unique; no two labels may have the same name, regardless of their scope.

You can designate up to 25 labels as “Top Labels”; these appear at the top of the list when a user clicks in a Label field.

Label Deletion
When label is deleted, all metrics data associated with the label are deleted.

Create Labels

  1. Select [Your name] > My Company > Labels.
  2. Click Add Labels.
  3. In the Label Name(s) field, enter one or more labels, separated by commas.
  4. Click Save.
  5. To designate a label as a Top Label, click the checkbox to the right of the label, in the Top Label column.

Scope a Label to an Initiative

  1. Select [Your name] > My Company > Labels.
  2. To the right of the label, click the pencil icon. Initiative-scoping must be enabled for this icon to appear. Click Assign Labels to Initiative at the top of the page to enable Initiative-scoping.
  3. Click the Initiatives drop-down menu.
  4. To allow any Initiative to use the label, select All Initiatives.
  5. To scope to a specific Initiative(s), select Individual Initiatives, and then check each Initiative.

Group a Set of Labels

You can create “Groups” of labels that are mutually exclusive, such that only one label from the group can be applied to an item at any given time (such as open/in process/closed; at any time, an item cannot be more than one of these). Label groups can not be scoped as group; each label must be scoped individually. Groups of labels can contain labels that are scoped differently from one another.

  1. Select [Your name] > My Company > Labels.
  2. Choose the labels to include in the group; click the checkbox to the left of each label.
  3. Click Create Group.
  4. In the Group name field, enter a name.
  5. Click Save.

You can replace labels with new ones. All content that had the original label is updated with the replacement label.

Replace a Label

  1. Select [Your name] > My Company > Labels.
  2. Choose the label to replace; click the checkbox to the left of the label.
  3. Click Replace
  4. In the Replace these label(s) on the same content field, enter the replacement label.
  5. Click Replace.

By default, users assign labels to content as part of the publishing or monitoring process (from within the publishing screens, Calendar and the Inbox). Users can also create new labels from those screens. To prohibit non-administrators from creating new labels, click the Restrict label creation to Company Administrators on this page checkbox. When using Initiative-scoped labels, this option changes from optional to required.

Crimson Hexagon Category Labels
When a user applies a Crimson Hexagon category to a stream item from the Inbox, the category is added as a label in , with the text “CH:” prepended to it. The label is added to even if the Restrict label creation to Company Administrators on this page is checked, and the user is a non-admin, because within the system, the categories already exist because of the Crimson Hexagon integration, and the label is only being “surfaced” by the user, not created. See Crimson Hexagon.

On the Labels tab, the Messages column shows the number of times the label has been applied to an outgoing message, company-wide (“Outbound”). The Stream Items column displays the number of times the label has been applied to an item in the Inbox (“Inbound”), company-wide.

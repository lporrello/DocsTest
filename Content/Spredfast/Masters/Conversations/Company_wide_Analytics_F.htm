Company-wide Analytics Filters

In Company-wide analytics, the default view of the dashboards contains data for all the Initiatives you have access to. A series of filters lets you narrow that large data set down so that you see only what you need. Filters in the Company Analytics dashboards are “sticky”—that is, the most recently used filter is remembered for each user.

When selecting individual items within a filter, you are limited to 500 items, such as 500 labels or 500 users.

Apply the date range filter first
Note that when you change a date range value, the page immediately refreshes using the new range. Set the correct date range before applying other filter criteria.

Initially, the Apply button is disabled on a page. When you begin to select criteria within a filter type (such as specific channels), the filter text appears in italics, indicating that criteria has been selected, but not applied. Select criteria for as many fields as you want, then click Apply. The following filters are available in Company Analytics:

  • Initiative—Select all Initiatives, or only specific ones. The list contains all Initiatives you have access to.

Company-wide Metrics
If you want to run reports on social media efforts company-wide, make sure you have access to all company Initiatives.

  • Account Set—Contains all Account Sets that are part of the Initiatives selected in the Initiative filter. Select all or only specific ones.
  • Channel—Contains all channels that have been authenticated in the Initiatives selected in the Initiative filter.
  • Account—Contains all accounts that are part of Initiatives selected in the Initiative filter. Select all or only specific ones. (Pinterest Boards are included as accounts.)
  • Date Range—Select the number of days to include. You can select a range of specific dates using the calendars. You can view data going back up to 13 months. Apply this filter first.
  • Label (Label 1)—Choose Label(s) that have been applied to outbound (published) messages. (For metrics related to Inbound labels (such as Customer Care metrics and reports on labels applied from the Inbox), see Initiative Analytics).
    For example: If you select “Customer Outreach” and “Marketing,” you will get messages that have either label applied to them.
  • Label 2—Use the Label 2 filter to make “&” (AND-type) queries with the Label 1 filter.
    For example: If you set Label 1 to “Customer Outreach” and “Marketing”, and Label 2 to “Whitepaper,” you would get messages that have:
    • Both Customer Outreach and Whitepaper
    • Both Marketing and Whitepaper
    • Customer Outreach and Marketing, and Whitepaper
  • User name—Select all or specific users. The list contains users who have at least one publishing record in the selected Initiative(s).
  • Targeting Profile—Choose Facebook targeting profiles that have been created and saved within . See Targeting Profiles.
  • Unpublished Page Posts (Facebook only. Also known as “Dark Posts.”)—Select Previously Unpublished Posts (For items that were originally created as dark posts, then subsequently published; these items are no longer counted as dark posts once they are published), Published Page Posts, or Unpublished Page Posts (Dark Posts).
    See Targeting.
  • Responses—Within , “responses” are items that are published in response to content that appears in Inbox streams. It can sometimes be helpful to exclude this type of publishing activity from performance analysis. To exclude responses, open the filter and deselect (uncheck) response. Then click Apply.
    The following are counted as Responses in :
    • Messages published using the “Amplify” feature in the Inbox
    • Twitter Replies and Direct Messages made from the Inbox
    • Facebook Private Messages (published through or auto-imported).
    • Twitter Retweets (both types, Editable and ‘native’) made from the Inbox
    • Comments made from the Inbox on Facebook, YouTube, LinkedIn, Weibo, Instagram, or Google+ Posts. Facebook and Weibo Replies are counted as Comments.
    • For Tweets published outside of , then auto-imported, considers the Tweet a “response” if the Tweet begins with “RT” or if Twitter identifies the Tweet as a response within the metadata it provides (the in_reply_to_status_id value is set).
    • Weibo reposts made from the Inbox

Company Analytics Filters can be saved and reused. Saved filters are available per Dashboard. (For example, a filter you create in the All Channels dashboard is available on all the tabs in the All Channels dashboard, but is not available on the Twitter dashboard.) Filters you create are not visible to anyone but yourself.

Reuse a Filter

  1. Select appropriate criteria, and then click Apply.
  2. In the upper right-hand corner of the page, in the Unsaved view drop-down menu, Unsaved View section, click Save.
  3. In the View name field, provide a name.
  4. Click Save. The new named filter appears in the upper right-hand corner. You can switch among saved views, rename them, or delete them.

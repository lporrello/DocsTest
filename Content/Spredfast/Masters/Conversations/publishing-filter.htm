Basic / Advanced Filter

The Calendar uses two filter mechanisms: Basic and Advanced.

  • Filter is inclusive, not exclusive. If you click Twitter, only Twitter items are returned in the results.
  • Filter searches across day/days displayed on the page; if you are looking at a week, the filter searches across the week.
  • Filters are additive. You can apply multiple criteria.
  • You can use Basic and Advanced filters together; selections you make and Apply in the Basic filter are remembered in the Advanced filter.
  • You can save, name, and reuse filters. From the Advanced filter screen, build a filter, click Save And Apply, and provide a name. The saved filter is per user, per Initiative, and also per screen in ; filters you create in the Calendar, for example, are not available in the Inbox, and filters you create in one Initiative are not available in another Initiative. Your list of saved filters appears in the Filters drop-down menu.
  • To edit a saved filter, from the Advanced filter dialog box, select the saved filter, make appropriate changes, and click Save and Apply.
  • Whenever a filter is on, the Filter On x label appears in the Calendar tool-bar. If you hover over the label, a popup describes the attributes of the current filter. Click the x to the right of the label to remove the filter.
  • When a filter has been applied, it remains until it has been removed, even when a user logs out and logs in again, or changes browsers.
  • The filter searches 180 days before and after the selected date (360 total days).

Basic Filters

The Basic filter includes the most commonly used filter criteria:

  • Initiative—select one, several, or all the Initiatives you have access to; items from these Initiatives appear with italicized names. If you want a company-wide Calendar, make sure you have access to all Initiatives.
  • Message status—Where the message is in the review/approval process.
  • "Starred" status—Items that have been “starred” by me in the Agenda
  • Assigned to me
  • Exclude Responses—"Response" includes all items scheduled from the Inbox, plus responses to Inbox items that were published outside of , then auto-imported.
  • The following are counted as Responses in :
    • Messages published using the “Amplify” feature in the Inbox
    • Twitter Replies and Direct Messages made from the Inbox
    • Facebook Private Messages (published through or auto-imported).
    • Twitter Retweets (both types, Editable and ‘native’) made from the Inbox
    • Comments made from the Inbox on Facebook, YouTube, LinkedIn, Weibo, Instagram, or Google+ Posts. Facebook and Weibo Replies are counted as Comments.
    • For Tweets published outside of , then auto-imported, considers the Tweet a “response” if the Tweet begins with “RT” or if Twitter identifies the Tweet as a response within the metadata it provides (the in_reply_to_status_id value is set).
    • Weibo Reposts made from the Inbox
  • Social channels—Blog, Facebook, Google+, LinkedIn, Twitter, Weibo, YouTube
  • Promoted—items that have a Facebook promotion created through associated with them (in any status). Appears only if Content Promotion has been enabled for your company.

Advanced Filters

The Advanced filter includes the options from the Basic filter, plus additional criteria:

  • User
  • Promoted Content—If the message has a Facebook promotion created through associated with it: No Promoted Content filter (Removes Promoted Content as a filter criterion), Promotions on all Published and Pending Messages, Promotions on Published Messages (original message has been posted), Active Promos (The Promotion is in Active status), Completed Promos (The Promotion is in Complete status), Promos pending FB Review (The Promotion is being reviewed by Facebook), Paused Promos (The Promotion is in Paused status), Promos with Errors (The Promotion is in Error status), Promos pending Billing info (The Promotion is pending Facebook Billing/Account issues), Failed Promos (The Promotion is in Failed status), Promotions on Pending Messages (The original message has not yet been published, so it has not been sent to Facebook for review).
  • Approval Team
  • Response—”Response” includes all items scheduled from the Inbox, plus auto-imported responses. By default responses are included; click Exclude responses to delete them from the display, or Responses only to show only scheduled responses in the display.
  • Account Set
  • Message type (blog, comment, event, message (microblogs and Pinterest pins), photo album, presentation, video) Note that In February, 2015, began using an updated version of the Facebook API (2.2) which no longer supports Facebook Notes (blogs), or Events. Information about these message types published prior to the change is still maintained in the platform, but no new data is available.
  • Account
  • Labels—By Outbound label (labels applied to outgoing content).
  • Publish type—How the message was created or discovered: as an Announcement (from a blog post, video, photo, discussion topic), Auto-imported (published on the native channel, then imported into ), Automated Retweet (a scheduled reposting of a published tweet by a different account), From Message Importer (message was uploaded using the Message Importer), Scheduled Content (microblog and non-microblog messages published through ), Show Unpublished Posts Only (the message was created as a "Dark Post") or Show Visible Posts Only (any message that was not created as an unpublished post).
    Note that within this filter, when you use the Auto Imported plus Unpublished Page Post filters, the results include items that meet both criteria. (That is, the results are “and”, not “or” in this one instance.)
  • Targeting type—Whether or not targeting was applied to the message.
